# Quiz MVP (Node.js + SQLite + Socket.IO)

Примитивный, но рабочий MVP веб-приложения для проведения квизов с ролями участника и организатора.

## Что реализовано

- Регистрация и авторизация пользователей с ролями `participant` и `organizer`
- Создание квизов организатором: категория, время на вопрос, правила
- Добавление вопросов:
  - текст вопроса
  - опциональная картинка (URL)
  - одиночный или множественный выбор
- Запуск комнаты по коду
- Подключение участников к активной комнате по коду
- Реалтайм-показ вопросов (Socket.IO)
- Ограничение по времени: ответ принимается только в течение показа вопроса
- Подсчет очков и общий лидерборд
- Личные кабинеты:
  - организатор: созданные квизы + история проведенных сессий
  - участник: подключение по коду + история участий

## Стек

- Backend: Node.js, Express
- Realtime: Socket.IO
- DB: SQLite (`better-sqlite3`)
- Frontend: HTML, CSS, vanilla JavaScript

## Как запустить

```bash
npm install
npm run start
```

После запуска:

- Откройте `http://localhost:3000`

## Автоматическая проверка (группа пользователей)

```bash
npm run test:e2e
```

Что делает тест:

- Поднимает сервер
- Регистрирует 1 организатора и 3 участников
- Проверяет, что участник не может создавать квиз
- Создает квиз и вопросы от организатора
- Создает комнату и подключает всех по WebSocket
- Запускает квиз, отправляет ответы от участников
- Проверяет финальный лидерборд и историю

## Дополнительные проверки качества

Smoke-прогон 15 кейсов:

```bash
npm run test:smoke
```

Нагрузочный мини-тест на 10-30 участников (по умолчанию 20):

```bash
npm run test:load
```

Пример запуска на 30 участниках:

```bash
$env:PARTICIPANTS=30; npm run test:load
```

Проверка граничных случаев:

```bash
npm run test:edge
```

Покрытие edge-прогона:

- пустые поля (валидация)
- повторные входы одним пользователем
- двойной клик по отправке ответа
- обрыв соединения и повторный вход в live-сессию

## Быстрый сценарий проверки

1. Зарегистрируйте пользователя с ролью `organizer`
2. Создайте квиз и добавьте в него вопросы
3. Нажмите `Создать комнату` и скопируйте код
4. В другом браузере/инкогнито зарегистрируйте `participant`
5. Подключитесь по коду комнаты
6. На странице комнаты у организатора нажмите `Старт квиза`
7. Участники отвечают, после завершения показывается лидерборд

## Структура

- `src/server.js` - API, авторизация, история, раздача страниц
- `src/socket.js` - realtime-логика комнат, вопросы, ответы, лидерборд
- `src/db.js` - инициализация SQLite и схема БД
- `public/` - клиентские страницы и JS-логика

## Важные ограничения MVP

- Нет восстановления live-таймеров после полного рестарта сервера
- URL картинок принимается как есть (без загрузки файлов)
- Без e2e-тестов и без прод-уровня hardening
