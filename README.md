# Пояснительная записка
## Веб-приложение для проведения квизов в реальном времени

---

## 1. Цель и предмет работы

Целью настоящей работы является разработка работоспособного прототипа (MVP) веб-приложения для проведения квизов в режиме реального времени с поддержкой двух ролей: организатора и участника. Приложение должно обеспечить возможность создания квизов, добавления вопросов, управления сессиями по коду комнаты и отображения результатов в режиме реального времени.

---

## 2. Обоснование выбора средств разработки

### 2.1 Серверная платформа: Node.js + Express

**Выбор обоснован:**
- **Быстрота разработки**: Node.js с Express позволяет быстро создавать REST API и обрабатывать HTTP-запросы. Для прототипа это критично, так как минимизирует время на настройку инфраструктуры.
- **Единый язык**: JavaScript на клиенте и сервере упрощает обмен данными и развертывание.
- **Встроенная поддержка асинхронности**: асинхронность встроена в язык, что упрощает работу с WebSocket и реальным временем.
- **Экосистема npm**: обширный репозиторий пакетов, включая Socket.IO для realtime-коммуникации.

### 2.2 База данных: SQLite

**Выбор обоснован:**
- **Простота развертывания**: не требует отдельного сервера баз данных, файловая БД подходит для разработки и тестирования.
- **Синтаксис SQL**: стандартный язык запросов, знаком разработчикам.
- **Достаточная производительность**: для MVP с ожидаемой нагрузкой на 10–30 одновременных пользователей достаточна.
- **Встроенная поддержка транзакций**: позволяет обеспечить консистентность данных при одновременном доступе.

Библиотека **better-sqlite3** выбрана для синхронных запросов с хорошей производительностью.

### 2.3 Realtime-коммуникация: Socket.IO

**Выбор обоснован:**
- **WebSocket поддержка**: обеспечивает двусторонний обмен данными в режиме реального времени между сервером и клиентами.
- **Fallback механизмы**: Socket.IO поддерживает fallback на HTTP-polling, что обеспечивает совместимость с более старыми браузерами и сетями.
- **Простота использования**: интуитивный API как на сервере, так и на клиенте.
- **Соответствие требованиям**: синхронизация вопросов, ответов и лидерборда требует именно такого уровня взаимодействия.

### 2.4 Клиентская часть: HTML, CSS, JavaScript

**Выбор обоснован:**
- **Минимальные зависимости**: отсутствие сложной сборки (webpack, babel) ускоряет разработку и упрощает развертывание.
- **Прямое управление DOM**: дает полный контроль над интерфейсом без абстракций.
- **Стабильность**: vanilla JavaScript отказывает в меньшем количестве ситуаций, чем сложные фреймворки.

## 3. Основные этапы разработки

### Этап 1: Подготовка и планирование (2 дня)
- Определение требований и функциональности.
- Проектирование схемы базы данных (таблицы для пользователей, квизов, вопросов, сессий, ответов).
- Выбор стека технологий.

### Этап 2: Инициализация проекта и инфраструктура (1 день)
- Создание Node.js проекта с Express.
- Настройка SQLite и инициализация схемы БД.
- Подготовка структуры папок (src/, public/, tests/).

### Этап 3: Реализация авторизации и ролей (2 дня)
- Регистрация и вход пользователей.
- Хеширование паролей (bcryptjs).
- Управление сессиями (express-session).
- Разграничение прав доступа для организатора и участника.

### Этап 4: API для управления квизами (2 дня)
- Создание квиза (POST /api/quizzes).
- Добавление вопросов (POST /api/quizzes/{id}/questions).
- Получение списка квизов (GET /api/quizzes/mine).
- Запуск сессии (POST /api/quizzes/{id}/start).

### Этап 5: Realtime-логика с Socket.IO (3 дня)
- Подключение к комнате (room:join).
- Показ вопросов с таймером.
- Прием ответов только в активный интервал.
- Подсчет баллов и обновление лидерборда.
- Завершение квиза и сохранение результатов.

### Этап 6: Интерфейс (3 дня)
- Страница авторизации (регистрация и вход).
- Личный кабинет организатора (создание квиза, добавление вопросов, список квизов).
- Личный кабинет участника (подключение по коду).
- Страница live-квиза (отображение вопроса, варианты ответа, лидерборд).
- История и результаты.

### Этап 7: Тестирование и документация (2 дня)
- Автоматизированное e2e-тестирование (4 пользователя).
- Smoke-прогон (15 кейсов).
- Нагрузочный тест (30 участников).
- Граничные случаи (пустые поля, разрывы соединения, двойная отправка).
- Написание README и рабочих инструкций.

### Этап 8: Оптимизация и доработка (1 день)
- Улучшение UX интерфейса (переразложение элементов кабинета).
- Исправление обнаруженных багов.
- Финальная проверка.

## 4. Реализованная функциональность

### Для организатора:
- Создание квиза с указанием названия, категории, времени на ответ и правил.
- Добавление вопросов (текстовые, с изображением).
- Поддержка одиночного и множественного выбора.
- Запуск комнаты по коду.
- Управление сессией (старт квиза).
- История проведенных сессий с результатами и победителями.

### Для участника:
- Регистрация и вход.
- Подключение к комнате по коду.
- Просмотр вопросов в реальном времени.
- Отправка ответов в активный интервал времени.
- История участия с очками и местом.

### Общее:
- Real-time синхронизация вопросов и ответов.
- Автоматический подсчет баллов.
- Лидерборд.
- Сохранение полной истории квизов и результатов.

---

## 5. Результаты тестирования

- **E2E-тест**: PASS (3 участника, корректные очки и финальный лидерборд).
- **Smoke-прогон**: PASS 15/15 (регистрация, роли, создание квиза, подключение, realtime, история).
- **Нагрузочный тест**: PASS на 30 участников (Duration: 15123 ms, Score range: 1..3).
- **Edge-cases**: PASS 4/4 (пустые поля, повторные входы, двойной клик, reconnect).

Приложение стабильно работает при одновременном подключении нескольких пользователей и корректно обрабатывает граничные случаи.

Структура репозитория:

- `src/db.js` — инициализация БД и схема.
- `public/` — HTML, CSS, JavaScript клиента.
- `tests/` — автоматизированные тесты (e2e, smoke, load, edge).

### C. Ссылка на работоспособный продукт
**Локальный запуск:**
```
npm install
npm run start
http://localhost:3000
```

**Демонстрационный сценарий:**
1. Зарегистрируйтесь организатором.
2. Создайте квиз и добавьте вопросы.
3. Нажмите "Создать комнату" и скопируйте код.
4. В инкогнито/другом браузере зарегистрируйтесь участником.
5. Подключитесь по коду комнаты.
6. На странице организатора нажмите "Старт квиза".
7. Участник отвечает, после завершения появляется лидерборд.

**Автоматизированная проверка:**
```
npm run test:e2e      # End-to-end (4 пользователя)
npm run test:smoke    # Smoke-тест (15 кейсов)
npm run test:load     # Нагрузочный (30 участников)
npm run test:edge     # Граничные случаи
```
